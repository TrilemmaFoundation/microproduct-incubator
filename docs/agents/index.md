Agents should treat [Build Trilemma](/) as the control plane for scaffolding and validating microproducts.

## Start here

- Root agent file (plain text): [`/AGENTS.md`](pathname:///AGENTS.md)
- Full compressed context bundle: [`/llms-full.txt`](pathname:///llms-full.txt)
- Product schema JSON: [`/schemas/product.schema.json`](pathname:///schemas/product.schema.json)
- Machine-readable registry JSON: [`/registry.json`](pathname:///registry.json)

## Workflows on this site

1. Pick an [archetype](/archetypes) closest to your user decision.
2. Choose a starter from [templates](/templates) (repo `product-templates/`).
3. Author the microproduct contracts in [Standards](/standards).
4. Register the shipped product metadata in [`/registry.json`](pathname:///registry.json) when ready for broader discovery.

See also the human playbook under [Documentation](/docs/intro/what-is-a-microproduct).
